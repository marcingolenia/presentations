Some Drawbacks:

Some Drawbacks:

  1. People won't care about terraform code provided by scaffolding. (A)
  2. It requires some maintenance both the "Ska" and templates (upgrading packages).
  3. It imposes a certain structure or architecture on the project, which may not always align with the specific requirements of the project
test note

And good things:

test note

And good things:

  1. Applying company-wide defaults related to infrastructure (like cloud armor policies) can be provided out of the box. (But for new projects only)
  2. Possibility to document the template with a explanation why we have in terraform the stuff we have. It may help the others to become more aware about the infrastructure.
  3. Rapid Development: Scaffolding provides a head start by generating the initial structure, configuration, and setup for a new project.
  4. You own the code

Todos:

  1. Add more templates including NodeJS
  2. Add more to "ignite go" template (like postgres + multitenancy or GRPC integrations)
  3. Integrate with git and add Web UI, deploy it internaly
  4. Make it possible to have templates in a dedicated Github repo, so the ska won't have them and can be still open source.
